# ТЗ — Esports.kz

> Платформа-сообщество для киберспорта Казахстана. Регулярные турниры по CS2, Dota 2, PUBG. Профили, команды, поиск тиммейтов, чат, трансляции, лидерборды, спонсорство.

**Статус:** черновик от 2026-05-01. Документ живой — правится по ходу обсуждений.

---

## 1. Идея и цель

Создать локальное (Казахстан → СНГ) сообщество киберспортсменов, где они могут:
- участвовать в регулярных чемпионатах с прозрачными правилами;
- находить тиммейтов, собирать команды, общаться;
- следить за турнирами, трансляциями и статистикой топ-игроков.

Эталон визуальной/смысловой подачи: **HLTV.org** (новости/расписание/статистика), **FACEIT** (организация матчей), **Liquipedia** (история турниров).

**Ключевое отличие от существующих платформ:** локальность (KZ), русскоязычный интерфейс, акцент на сообществе и игроках, а не на профессиональной сцене.

## 4. Функциональные модули

### 4.1. Аккаунты и профили

- Регистрация и вход через **Steam OpenID** (бесплатно, идеально для геймеров).
- E-mail только как дополнительный канал связи (не обязателен).
- Поля профиля:
  - ник, аватар (из Steam), регион/город;
  - **теги дисциплин** — мульти-выбор: CS2, Dota 2, PUBG (можно все три);
  - игровая роль (для CS2: AWP/Rifler/IGL/Support; для Dota 2: Carry/Mid/Offlane/Support; для PUBG: Fragger/Sniper/IGL/Support);
  - часовой пояс, доступное для игр время (вечер будни / выходные и т.п.);
  - био (свободный текст);
  - Twitch/Discord-теги (опционально);
  - статус: онлайн / ищет тиммейта / в матче / оффлайн.
- Публичная страница профиля: статистика турниров, команда, MVP-награды.

### 4.2. Команды

- Игрок может состоять в **одной команде на дисциплину** (т.е. одновременно в CS2-команде и Dota-команде — да; в двух CS2-командах — нет).
- Капитан создаёт команду: имя, тег (3-5 символов), лого, регион, дисциплина.
- Приглашение через ссылку или поиск по нику.
- Командная страница: состав, история матчей, рейтинг, призовые.

### 4.3. Поиск тиммейтов (LFG — Looking For Game)

- Лента анкет: "ищу команду CS2, AWPer, MM 18к, Алматы, вечером".
- Фильтры: дисциплина, роль, регион, ранг/опыт, доступное время, онлайн прямо сейчас.
- Кнопка "написать в чат" — открывает 1-на-1 диалог.
- Капитаны могут вешать "ищем 1 игрока на роль X".

### 4.4. Турниры и матчи

**Форматы:**
| Дисциплина | Формат MVP |
|---|---|
| CS2 | **Double Elimination**, BO1 (групповой) → BO3 (плейофф) → BO5 (финал) |
| Dota 2 | **Single Elimination**, BO2 (группа) → BO3 (плейофф) → BO5 (финал) — уточнить с комьюнити |
| PUBG | **Серия из N матчей**, очковая система (kill points + placement) |

**Жизненный цикл турнира:**
1. Админ создаёт турнир: название, дисциплина, даты, формат, призовой фонд, требования к командам.
2. Открывается приём заявок от команд.
3. Админ закрывает заявки → автоматически генерируется сетка.
4. Команды играют свои матчи **на сторонней платформе** (FACEIT / Cybershoke / FastCup) по согласованным правилам — лобби создаёт админ или капитаны по инструкции.
5. После матча капитан загружает скриншот результата, админ подтверждает.
6. Сетка обновляется автоматически.
7. После турнира — выдача наград, обновление лидерборда, MVP-голосование (v2).

**Страница турнира содержит:**
- описание, правила, призы;
- сетка (визуал bracket);
- участвующие команды;
- расписание матчей с временем и встроенным **Twitch iframe**;
- лента результатов;
- статистика (после турнира).

### 4.5. Трансляции (Twitch integration)

- На страницах турниров и матчей — встроенный плеер Twitch (iframe).
- Главная: блок "LIVE сейчас" с активными стримами зарегистрированных игроков.
- Игрок может в профиле указать свой Twitch-канал.
- Для платформы это **бесплатно** — Twitch iframe API публичный.

### 4.6. Чат и социальные функции

- **Личные сообщения** 1-на-1.
- **Командный чат** для участников команды.
- **Турнирный чат** для участников конкретного турнира (опционально на MVP).
- Друзья: запросы, подтверждение, лента онлайн-друзей.
- Поиск по тегам и онлайн-статусу.
- Уведомления: приглашение в команду, начало матча, новое сообщение.

### 4.6.1. Статистика игроков (HLTV-style)

**Эталон:** https://www.hltv.org/stats/players/24144/molodoy

**Что отслеживаем:**

*Базовые показатели (общие для всех игр):*
- Total kills / deaths / assists
- MVPs (раундов)
- Rating 2.0 (HLTV-формула, рассчитывается автоматически из других метрик)
- Maps / matches played

*CS2-специфика:*
- ADR (Average Damage per Round)
- Headshot %
- KAST (% раундов с Kill/Assist/Survival/Trade-kill)
- Opening kills / opening deaths
- Clutches 1v1, 1v2, 1v3+
- Utility damage, flash assists
- Per-map performance (Mirage, Inferno, Anubis...)

*Dota 2-специфика:*
- GPM (Gold per minute) / XPM
- Last hits / denies
- Hero damage, tower damage
- Healing
- Hero pool / win rate per hero

*PUBG-специфика:*
- Headshot kills, longest kill
- Damage dealt, distance traveled
- Average placement
- Squad kills / solo kills

**Уровни агрегации:**
- По матчу — `MatchPlayerStat` (1 запись на игрока в матче)
- По турниру — агрегируется на лету
- По сезону (Spring 2026, Summer 2026...) — `Season` группирует турниры
- По всему времени — все матчи игрока

**Кто вводит:**
- На MVP: админ вручную после матча, через форму "Ввести статистику матча".
- В v2: попытка автоматической интеграции с FACEIT API для CS2.
- В v3: парсинг demo-файлов CS2 / replay Dota 2 / replay PUBG (опционально, сложно).

**Страница игрока:**
`/players/[username]/stats`
- Сводка: общий рейтинг, K/D, ADR, HS%, маппы
- Графики: рейтинг по времени, форма за последние 10 матчей
- Топ-карты, лучшие/худшие оппоненты
- Список последних матчей со статой

### 4.6.2. Жеребьёвка и сетки турниров

**Форматы:**

*Double Elimination (CS2):*
- Upper Bracket (UB) → проигравший падает в Lower Bracket (LB)
- LB → победитель идёт в Grand Final
- Grand Final: bracket reset (UB-чемпион имеет 1 жизнь дополнительно)
- Жеребьёвка: random или seeded по rating команд

*Single Elimination (Dota 2):*
- Один проигрыш = вылет
- Часто с групповым этапом перед плейофф

*Round Robin (групповой этап):*
- Каждый играет с каждым в группе
- По итогам — топ-N выходят в плейофф

*Battle Royale Series (PUBG):*
- N матчей подряд, очки за позицию + kill points
- По сумме — топ команд

**Реализация:**
- Расширили `Match` полями: `round`, `bracketSide` (UB/LB/GROUP), `bracketPosition`, `parentMatchAId/B`.
- Алгоритм генерации сетки:
  - Для DE: 1 раз "посеять" команды → создать матчи UB1, UB2 ... → создать LB-матчи с правильными `parentMatch*` ссылками.
  - Реализовать в админке: кнопка "Сгенерировать сетку" после закрытия регистрации.
- Визуализация:
  - Страница `/tournaments/[slug]/bracket` с интерактивной сеткой (слева UB, справа LB, финал по центру).
  - Используем простую CSS Grid + SVG-линии. Без сторонних bracket-библиотек на MVP.

**Жеребьёвка-лотерея:**
- Опционально (v2) — анимация "Live Draw" в браузере админа: с эффектом case-opening, лого команд "крутится", выпадает в нужный слот.
- Можно стримить на Twitch перед турниром.



- **TOP PLAYERS** — лучшие игроки сезона (по рейтингу).
- **BEST SNIPERS** — AWP-короли в CS2 (по K/D с AWP, статистика вручную или через API FACEIT в v2).
- **CHAMPIONS** — победители завершённых турниров.
- **MVP** — самый ценный игрок турнира.

⚠ **Важно:** все лидерборды наполняются **только админом** вручную (на MVP). Никаких автоматических расчётов из match-данных в первой версии.

### 4.8. Спонсорство и партнёры (новое — добавлено по запросу)

**Концепция:** ежемесячная подписка для брендов, желающих поддержать киберспорт Казахстана и получить охват среди целевой аудитории (геймеры 16-35).

**Тиры спонсорства:**

| Тир | Что входит | Цена/мес (черновик) |
|---|---|---|
| **БРОНЗА** (Жез) | Лого в подвале, упоминание в одном турнире/мес | от 50 000 ₸ |
| **СЕРЕБРО** (Күміс) | Лого + плашка на главной, бренд в 2 турнирах/мес, пост в соцсетях | от 200 000 ₸ |
| **ЗОЛОТО** (Алтын) | Title-спонсор турнира раз в квартал, баннер на стримах, наградной кубок имени бренда | от 500 000 ₸ |
| **ПЛАТИНА** | Эксклюзив на сезон, ко-брендинг, доступ к аудитории, кастомная активация | по запросу |

**Why это важно:** монетизация без необходимости брать комиссию с игроков, и в дополнение — даёт легитимности проекту перед локальными брендами (Beeline KZ, Kaspi, банки, gaming-периферия, букмекеры — где допустимо по законодательству).

### 4.9. Маркет скинов — ОТЛОЖЕНО ⛔

Изначальная идея: сайт как нейтральная сторона для обмена и продажи скинов CS2 за деньги, с комиссией за эскроу.

**Причины отложить на v3+ (или не делать вообще):**

1. **Регуляторные риски в РК**: в Казахстане деятельность по проведению расчётов между физлицами требует лицензии (если оборот превышает пороги, попадает под закон о платёжных услугах № 11-VI). Эскроу-сервисы за реальные деньги — отдельная юридическая категория.
2. **Steam ToS**: Valve запрещает продажу предметов Steam за реальные деньги вне их собственного маркета. Аккаунты продавцов могут блокироваться. Это создаёт юридический риск для нашей платформы (претензии от пользователей, чьи аккаунты заблокировали).
3. **Платёжные провайдеры**: ЮKassa/Stripe/Kaspi-Pay могут отказать в обслуживании категории "виртуальные предметы / скины".
4. **AML/KYC**: финансовые сервисы обязаны проводить идентификацию клиентов. На MVP это перегруз.
5. **Сложность разработки**: эскроу-логика + интеграция с Steam Trade API + защита от мошенничества — это отдельный продукт.

**Альтернативы, которые можно рассмотреть в v2:**
- Доска объявлений "обмен скинами" между пользователями, **без участия платформы в расчётах** (как форум). Risky, но юридически чище.
- Партнёрство с уже существующим маркетом (DMarket, CS.MONEY), реферальные ссылки.

**Решение пользователя:** отложить, не блокировать MVP.

## 4.10. Референсы и позиционирование

Чтобы не изобретать с нуля и не повторять чужих ошибок, мы изучили существующие платформы и выбрали, что брать и от чего отказаться.

### QCL.kz — Qazaqstan Counter-Strike League

Официальный чемпионат Казахстана по CS2. Партнёр Федерации киберспорта Казахстана (QCF).

**Что есть у них:**
- Регулярная лига по CS2 (13 туров).
- Призовой фонд 500 000 ₸ за сезон.
- Региональное представительство (команды со всех областей).
- Трансляции с профессиональными комментаторами.
- Рейтинг команд и игроков.
- Новости.

**Что мы берём:**
- Идею регулярного турнирного цикла (не разовые турниры, а серия → финал).
- Формат региональных команд.
- Партнёрство с QCF в перспективе (после первого успешного сезона).

**Что мы делаем иначе/лучше:**
- **Три дисциплины** вместо одной (CS2 + Dota 2 + PUBG).
- **Open community**: не только профи-команды, любительский слой важнее по объёму.
- **LFG / поиск тиммейтов** — у QCL этого нет, у нас это центральная фича.
- **Профили и социальная часть** — друзья, чат, теги дисциплин, MVP-награды.
- **Современный UI** — у QCL дизайн новостного портала, у нас sci-fi/неон.
- **Прозрачные тиры спонсорства** — у QCL партнёрство непрозрачное, у нас публичная сетка цен.

### HLTV.org — мировой эталон CS-новостей

Глобальный портал CS2-новостей, статистики, рейтингов. Существует с 2002 года.

**Что есть у них:**
- 3-колоночный layout: матчи слева, новости в центре, рейтинги/события справа.
- Высочайшая информационная плотность: можно охватить десятки матчей на одном экране.
- Глубокая статистика игроков (Rating 2.0, K/D, ADR, KAST, headshot %).
- Расписание матчей, результаты, форум, fantasy-режим, betting-разделы.
- Цветовые маркеры статусов: LIVE / FINISHED / UPCOMING.

**Что мы взяли:**
- **3-колоночный layout** на главной (матчи слева — лента в центре — рейтинги справа).
- **Sticky-колонки** для расписания и рейтингов.
- **Информационная плотность** — много данных, а не воздуха.
- **Цветовые маркеры статусов** матчей.
- **Top Teams / Top Players** ranking блоки.
- **Featured Tournament** виджет.

**Что мы улучшили:**
- **Sci-fi эстетика** вместо устаревшего дизайна 2010-х (clip-corner кнопки, неон, моноширинные акценты, scan-line анимации).
- **Live-тикер наверху** — мгновенное цветное уведомление об идущих матчах (у HLTV такого нет).
- **Тренд-индикаторы** в Top Teams (+24/-12) с цветовой индикацией.
- **Спонсорский слот** прямо в правой колонке — монетизация в каждом просмотре.
- **Кнопка прямой регистрации команды** на Featured Tournament card — у HLTV это отдельные страницы.
- **Featured news card** с фоновыми градиентами вместо плоских превью.
- Mobile-first: всё схлопывается в одну колонку без потери смысла.

### FACEIT — что взять, чему не подражать

Глобальная платформа для проведения матчей CS2/Dota 2 со встроенным анти-читом.

**Что мы берём идеологически:**
- LFG-система (поиск тиммейтов).
- Командные лобби.
- Система рейтинга / уровней.

**Что НЕ берём:**
- Сами матчи проводятся **на FACEIT**, мы не дублируем их инфраструктуру (это нерентабельно для MVP).
- Античит — у нас нет своего, полагаемся на стороннюю платформу.
- Система пинга / антипинг-серверов — это на FACEIT.

### Liquipedia — для вдохновения по статистике

Wiki-проект по киберспорту со структурированными страницами турниров, команд, игроков.

**Что взять в v2:**
- Структура страниц «Турнир» (формат, призовой, участники, сетка, результаты, MVP).
- Структура страниц «Игрок» (история команд, трофеи, статистика).

### Формула проекта

```
HLTV (информационный слой и плотность)
+ QCL (турнирный движок и серьёзность)
+ FACEIT-lite (community: профили, LFG, чат)
+ Liquipedia (структурированная история игроков и команд) — v2+
+ Sci-fi UI (отличие от всех вышеперечисленных)
= Esports.kz
```

Уникальное позиционирование: **«Если QCL — это профи-лига для топ-10 команд, то Esports.kz — это весь остальной киберспорт Казахстана: любители, полупрофи, поиск команд, обмен опытом, локальные чемпионаты.»**

## 6. Технологический стек

| Слой | Технология | Зачем |
|---|---|---|
| Фронт+бэк | **Next.js 16 (App Router) + TypeScript** | Один фреймворк, SSR, серверные actions |
| Стили | **Tailwind CSS 4** | Скорость прототипирования |
| База данных | **PostgreSQL (Supabase)** | Бесплатный тир, удобный UI, real-time |
| ORM | **Prisma** | Типобезопасные запросы, миграции |
| Авторизация | **NextAuth (Auth.js) со Steam-провайдером** | Steam OpenID — стандарт для геймеров |
| Real-time | **Pusher Channels** или **Supabase Realtime** | Чат и онлайн-статус |
| Хостинг | **Vercel** (фронт) + **Supabase** (БД) | Бесплатные тиры на старте |
| Платежи (v3) | **Kaspi.kz API** + ЮKassa | KZ-локальный + СНГ |
| Email | **Resend** | Бесплатный тир, простой API |
| Файлы (аватарки/скрины) | **Supabase Storage** или **UploadThing** | Бесплатно на старте |
| Аналитика | **Vercel Analytics** + **PostHog** | Бесплатные тиры |

**Вне MVP:**
- Мобильное приложение — отложено. Сайт делается mobile-first, этого достаточно.
- Native-чат (только web). В v2+ можно подумать про PWA или Telegram-бота.

## 7. Риски и ограничения

| Риск | Митигация |
|---|---|
| Не получим API-доступ к FACEIT | На MVP — ручной ввод результатов админом |
| Cybershoke / FastCup не дают API | Турниры играются по ссылкам, мы только ведём сетку |
| Маркет скинов = юридические проблемы | **Отложено на v3, может не быть реализовано** |
| Низкое привлечение пользователей | Стратегия: 1-2 турнира с призовым фондом → виральность; партнёрство с локальными стримерами |
| Спонсоры не приходят без аудитории | Курица-яйцо — на старте сами организуем 2-3 турнира за счёт основателя для получения первого трафика |
| Steam OpenID требует свой application key | Получается за час на steamcommunity.com/dev — не блокер |
| Twitch iframe — в чате может быть мат/реклама | На MVP отдаём как есть; в v2 — можем заменить на встраиваемый плеер без чата |

## 8. Открытые вопросы

- [ ] **Название проекта** — пока заглушка `Esports.kz`. Нужно решить и проверить домен.
- [ ] **Регистрация юрлица** — для приёма платежей от спонсоров и оплаты призовых. ИП или ТОО?
- [ ] **Первый призовой фонд** — кто финансирует? Из своих, краудфандинг, договорённость со спонсором заранее?
- [ ] **Договор со спонсорами** — нужна юр. форма (вероятно публичная оферта + индивидуальные доп. соглашения).
- [ ] **Модерация** — кто будет модератором на старте (мат, оскорбления, читы)? Один админ — bottleneck.
- [ ] **Анти-чит на турнирах**: полагаемся на стороннюю платформу (FACEIT) или требуем стрим с лица?
- [ ] **Возрастной порог** для игроков — 14, 16, 18? Влияет на регуляторику (несовершеннолетние + соглашения).
- [ ] **Призовые: денежные или скинами/мерчем?** Денежные = налоги, договоры с победителями.
- [ ] **Турнирный взнос** с команд (entry fee) — да/нет? Может профинансировать призовой, но усложняет регуляторно.
- [ ] **Социальные сети** — какие веду параллельно (Instagram, TikTok, Telegram-канал)?
