# ТЗ — Античит-стратегия

> Что мы можем и что не можем делать с защитой от читеров.
> Реалистичный взгляд на возможности и компромиссы.

**Статус:** черновик от 2026-05-01.

---

## TL;DR

Свой полноценный античит уровня VAC / Vanguard / Easy AntiCheat — **не делаем**.
Это R&D отделы корпораций (Riot, Valve, Epic) с десятками инженеров и
10+ лет накопленного опыта.

**Наша стратегия:**
- **Сейчас**: турнирные матчи играются в FACEIT-комнатах с их античитом
- **v2 (через 6-12 мес)**: ML-анализ `.dem` файлов после матчей
- **v3 (опц.)**: партнёрство с готовым античитом (BattlEye/EAC) — от $100k/год

---

## Уровни защиты

### 1. **VAC (Valve Anti-Cheat)** — стандарт CS2
- ✅ Бесплатно, встроен в игру
- ⚠ Лояльно к новым читам (детектит сигнатуры старых чит-движков)
- 📊 Эффективность: ~30-40% от продвинутых читов

### 2. **CS2 Match-Anti-Cheat** (опция в game-серверах)
- Включается через `mp_match_anti_cheat 1` на сервере
- Строже чем VAC
- 📊 Эффективность: +10-20% к VAC

### 3. **FACEIT Anti-Cheat (FAC)**
- Платформа FACEIT использует свой клиентский античит
- Игроки скачивают и устанавливают локально
- Конфликтует с VAC, BattlEye — играть нужно отдельно
- 📊 Эффективность: ~70-80% против обычных читов
- ✅ **Это то, что мы используем для турнирных матчей**

### 4. **Демо-анализ + ML** (планируем для v2)
- После матча скачиваем `.dem` файл
- Парсим через `demoinfocs-golang` или `awpy`
- ML-модель оценивает подозрительные паттерны:
  - Reaction time < 100ms — быстрее человеческой реакции
  - Aim snaps — резкие незаконные движения мыши
  - Pre-fire через стену — стрельба точно в позицию противника без линии видимости
  - Walls — поведение указывающее на знание позиции противника
  - Recoil pattern recognition (RCS-detection)
- Результат: probability score 0..1
- Если > 0.85 → флаг для ручной модерации
- 📊 Эффективность: ~50-60%, требует постоянного дообучения

### 5. **Свой kernel-mode driver**
- Ring 0 драйвер, сканирует процессы и память игры
- ❌ Требует подписи Microsoft EV-сертификатом ($200-400/год)
- ❌ Конфликты с другими kernel anti-cheats (Vanguard, BattlEye, EAC)
- ❌ Риск BSOD у пользователей при ошибке
- ❌ Команда из 5+ инженеров на 1-2 года
- ❌ Постоянная гонка (кейтеры обновляются раз в неделю)
- **Не для нашего масштаба.**

### 6. **Hardware-level (DMA-detection)**
- DMA-карты (Direct Memory Access) — самый продвинутый чит. Читают память игры через PCIe минуя ОС
- Детекция требует специфических аппаратных проверок
- Только для топ-турниров с физическим присутствием
- Стоимость детекции: $$$$

## План для Esports.kz

### Phase 1 — MVP (сейчас)

**Решение:** все турнирные матчи проводятся в FACEIT-комнатах через их API.

**Поток:**
1. Админ создаёт турнирный матч на нашей платформе
2. Через FACEIT API создаётся match-room с включённым FAC
3. Капитанам отправляется invite-link на FACEIT
4. Игроки заходят, играют
5. После матча — наш парсер тянет статистику с FACEIT API

**Античит:** делегирован FACEIT.

**Плюсы:**
- 0 разработки античита
- Проверенная платформа
- Игроки уже привыкли к FAC

**Минусы:**
- Зависим от FACEIT (если они упали — наши турниры тоже)
- Нет данных кроме того что они отдают через API
- Платно при больших объёмах (FACEIT берёт комиссию с организаторов на масштабе)

### Phase 2 — Демо-анализ (через 6-12 мес)

**Цель:** дополнительный слой проверки матчей, играющихся не на FACEIT (например, в наших собственных серверах в Алматы — см. `TZ_SERVERS.md`).

**Технически:**
1. После каждого матча сервер сохраняет .dem
2. Background job парсит файл (Python + `awpy` библиотека)
3. Извлекает per-tick данные: позиция игрока, движения мыши, стрельба
4. ML-модель анализирует паттерны, выдаёт score
5. Score > 0.85 → отметка в БД "review needed"
6. Админ ручно проверяет матч

**Стек:**
- Python + awpy для парсинга
- scikit-learn / PyTorch для модели
- Или OpenAI GPT-4 для high-level analysis (дороже)
- Background queue (Vercel Cron + Supabase или отдельный worker)

**Время разработки:** 3-6 месяцев (одиночная разработка), быстрее с ML-инженером.

### Phase 3 — Опционально

**Если бизнес растёт и нужны турниры с большими призовыми:**
- Лицензия BattlEye / Easy AntiCheat — от $100k/год + per-CCU pricing
- Это уровень Riot Games / EA

**Реалистично для нас в ближайшие 2 года?** Нет.

## Что мы НЕ делаем

| Запрос | Почему не делаем |
|---|---|
| Свой kernel-driver | Слишком сложно, 1-2 года команды, конфликты |
| Memory scanner в браузере | Web ничего не может прочитать у игры |
| Капчу на каждый раунд | Убьёт UX |
| Ban-by-IP | Easily bypassed через VPN |
| Реал-тайм aim-detection без ML | Невозможно из веб-приложения |

---

## Что МОЖНО сделать прямо сейчас (без разработки античита)

1. **Запись демок обязательна** — все турнирные матчи должны записываться
2. **Public profile reveiw** — игроки могут пожаловаться, ссылка на демку, админ смотрит
3. **Reputation system** — игроки голосуют после матча "подозрителен" / "ок"
4. **Cooldown банов** — если 3+ жалоб за неделю → автоматически в "review queue"
5. **Strikes & Ban система** — 3 страйка → бан на 30 дней, 5 → permaban
6. **Stream обязателен в финалах** — игрок шарит экран на Twitch

Эти меры **дешевле** разработки античита, но дают **неплохое покрытие**
для community-уровня. Этого достаточно для KZ-сцены на старте.

## Решение

**Античит = FACEIT, наша платформа = матчмейкинг + статистика + сообщество.**

Если игрок захочет читерить — он попадётся на FACEIT-античите при играх в их лобби.
Если играет на наших серверах в Phase 2 — поймает ML-демо-анализ.
В крайнем случае — community reports + админ-ban.

**Этого достаточно для масштаба сообщества Казахстана** в первые 2-3 года.
